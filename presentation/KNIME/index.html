KNIME

  • The Konstanz Information Miner
  • Open source framework for data mining
  • Written in Java, its graphical workflow editor is implemented as an Eclipse plug-in => cross-platform
  • GUI
  • Intuitively drag and drop

P.S. Eclipse is famous for its Java IDE and an impressive collection of add-on tools for software developers

Applications

  • Provide a wide set of functionalities for many different common tasks, such as following:
    • Data mining
    • Cheminformatics
    • Image analysis
    • Text mining
    • Deep sequencing
      ...

Basic functionalities

  • Can be applied for many different type of analysis, such as following:
    • Input/Output
    • Data processing
    • Statistics
    • Data mining
    • Visulizaiton
      ...

Let's take a look

Brief Concepts

  • Basic components
    • Node: data processing unit
    • Edge: data flow, can branch at any point
  • The constructed workflow can be treated as a block diagram

Practical Aspects

  • Basic data manipulations are implemented as built-in nodes
  • As KNIME is not focused on data analysis of high-thriughtput experiments, it lacks common methods necessary for screen mining
  • Fortunately, communities have developed many useful extensions
    • RDKit
    • Vernalis
    • CDK
    • Indigo
      ...

Practical Aspects (cont'd)

  • If the worst comes to the worst, we can also develop custom nodes
  • As far as I know, there are three ways to acheive this:
    • Natively, extend the basic Node classes in Java
    • Develope the node under R, use the built-in node to execute R scripts
    • Utilize the scripting integration framewirk for KNIME (recommended) link
      *supported programming languages: R, python, Matlab, Groovy
